Fare prediction¶

1 Problem Statement¶

The predicting of fare amount (inclusive of tolls) for a taxi ride, given the pickup and dropoff locations,the pickup date time and many other attributes given below. The description of all the attributes is given below. The target variable is “fare_amount “

Understand the data and do necessary data exploration and try creating new features and build a machine learning model to predict the fare amount.

Variable

Description

TID

Unique ID

Vendor_ID

Technology service vendor associated with cab company

If a new user is taking the ride

toll_price

toll tax amount

tip_amount

tip given to driver (if any)

applicable tax

time at which the ride started

Using the WGS84 datum, this point is located in international waters in the Gulf of Guinea, in the Atlantic Ocean, off the coast of West Africa.

3.3 Creating new attributes distance from pickup and dropoff latitudes and longitudes¶

Euclidean Distance works for the flat surface like a Cartesian plain however, Earth is not flat. So we have to use a special type of formula known as Haversine Distance.

In [558]:
import haversine as hs
distance_in_meters = []
for i in range(data.shape[0]):
    pick=(data['pickup_longitude'][i],data['pickup_latitude'][i])
    drop=(data['dropoff_longitude'][i],data['dropoff_latitude'][i])
    distance_in_meters.append(hs.haversine(pick,drop,unit='m'))
data['distance_in_meters'] = distance_in_meters

Alternative Approach to calculate the distance travelled by a taxi which is more accurate than the Haversine Distance because the distance is calculated by open street map's open source api which calculates the route taken to reach the destination by car. But this approach is time consuming as well.
